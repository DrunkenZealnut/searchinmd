# Plan: SearchInMD 기능 확장 제안

## Executive Summary

| 항목 | 내용 |
|------|------|
| Feature | SearchInMD 프로젝트 기능 확장 로드맵 |
| 작성일 | 2026-04-03 |
| 현재 상태 | 키워드 검색 + 페이지 전체 내용 + 등급 분류 + 대시보드 완성 |

| 관점 | 설명 |
|------|------|
| **Problem** | 현재 키워드 검색과 등급 분류는 완성되었으나, 분석 결과의 활용/공유/심화 분석이 수작업에 의존 |
| **Solution** | 검색 고도화, 분석 자동화, 협업/공유, 데이터 파이프라인 4가지 축으로 기능 확장 |
| **Function UX Effect** | 검색→분석→보고서→공유까지 원스톱 워크플로우 완성 |
| **Core Value** | NCS 교재 안전보건 연구의 전체 생산성을 높이고, 재현 가능한 분석 체계 구축 |

## A. 검색 고도화 (Search Enhancement)

### A1. 복합 키워드 검색 (AND/OR/NOT)
- **현재**: 시트 이름 1개 = 키워드 1개로 단순 매칭
- **제안**: `"화학물질 AND 보호구"`, `"폭발 OR 화재"`, `"안전 NOT 유의사항"` 형태의 복합 조건 지원
- **가치**: 단일 키워드 검색으로는 놓치는 맥락적 연관성을 발견
- **난이도**: 중 (검색 로직 변경, UI에 조건 빌더 추가)

### A2. 유사어/동의어 자동 확장
- **현재**: 정확한 문자열 일치만 검색
- **제안**: "MSDS" 검색 시 "물질안전보건자료"도 자동 포함, "보호구" 검색 시 "마스크/안전모/보안경" 등 하위 개념 포함
- **가치**: 동의어를 모르는 연구자도 누락 없이 검색 가능
- **난이도**: 중 (동의어 사전 구축 필요, JSON 매핑 테이블)

### A3. 문맥 기반 검색 (LLM 강화)
- **현재**: LLM 하이브리드 검색은 문서 구조 파악에만 사용
- **제안**: "이 페이지가 안전 교육에 적합한가?"를 LLM에게 판단하게 하여 키워드 없이도 안전 관련 내용 탐지
- **가치**: 키워드에 의존하지 않는 의미 기반 분류로 등급 정확도 향상
- **난이도**: 상 (프롬프트 설계, 비용, 정확도 검증 필요)

---

## B. 분석 자동화 (Analysis Automation)

### B1. 자동 등급 분류 정확도 개선
- **현재**: 키워드 카운트 기반 (안전 키워드 ≤5 → 등급1, 조치 키워드 ≥5 → 등급3)
- **제안**: 문장 단위 분류 모델 적용. "안전에 유의한다" vs "방독면을 착용하고 환기 시설을 가동한 후 작업한다"를 구분
- **가치**: 현재 등급2(기본 언급)의 형식적 문구와 실질적 내용을 구별
- **난이도**: 중~상 (분류 규칙 고도화 또는 LLM few-shot 활용)

### B2. 교재간 비교 분석
- **현재**: 교재별 안전 비율만 산출
- **제안**: 같은 주제(예: 식각 공정)를 다루는 교재들의 안전 내용을 비교하여, 모범 교재와 부실 교재를 식별
- **가치**: "이 교재는 A 교재의 안전 내용을 참고하여 보강해야 합니다" 형태의 구체적 권고 가능
- **난이도**: 중 (교재 주제 매핑 + 비교 로직)

### B3. 시계열 변화 추적
- **현재**: 단일 시점 분석
- **제안**: 교재 개정판별 안전 내용 변화를 추적. "23v5 → 23v6에서 안전 내용이 추가/삭제되었는가?"
- **가치**: 교재 개정 시 안전 내용이 후퇴하는 것을 감지
- **난이도**: 중 (버전별 파일 관리, diff 로직)

## C. 결과 활용/공유 (Output & Collaboration)

### C1. 자동 보고서 생성 (PDF/DOCX)
- **현재**: Excel Export만 가능. 분석 보고서는 수동 작성
- **제안**: 대시보드의 통계 + 문제점 + 개선안을 정형화된 보고서(PDF/DOCX)로 자동 생성
- **가치**: 연구 결과를 즉시 보고서 형태로 제출 가능
- **난이도**: 중 (Python reportlab 또는 docx 라이브러리)

### C2. 대시보드 Excel 업로드 기능
- **현재**: 대시보드 데이터가 HTML에 하드코딩
- **제안**: 대시보드 페이지에서 Excel 파일을 업로드하면 자동으로 통계 생성 및 차트 렌더링
- **가치**: 새로운 분석 결과가 나올 때마다 코드 수정 없이 대시보드 갱신
- **난이도**: 중 (SheetJS로 브라우저에서 Excel 파싱 → Chart.js 업데이트)

### C3. 인터랙티브 교재 뷰어
- **현재**: 검색 결과에서 페이지 내용을 텍스트로만 표시
- **제안**: 마크다운 파일을 브라우저에서 렌더링하여 원본에 가까운 형태로 열람. 키워드 하이라이트 포함
- **가치**: Excel에서 나가지 않고 원본 맥락을 바로 확인
- **난이도**: 하~중 (marked.js로 마크다운 렌더링)

## D. 데이터 파이프라인 (Data Pipeline)

### D1. PDF→Markdown 변환 자동화
- **현재**: 별도 도구(pinecone_agent 등)로 변환 후 수동으로 폴더에 배치
- **제안**: SearchInMD에서 PDF 폴더를 지정하면 자동으로 Markdown + `_meta.json` 생성
- **가치**: PDF 투입부터 분석까지 원스톱 파이프라인
- **난이도**: 상 (OCR/PDF 파싱 라이브러리 통합)

### D2. `<!-- page: N -->` 마커 자동 삽입
- **현재**: TODOS에 있는 항목. 상류 파이프라인에서 마커를 삽입해야 함
- **제안**: 기존 마크다운 파일에 `_meta.json` 기반으로 페이지 마커를 후처리로 삽입하는 스크립트
- **가치**: DP 기반 페이지 매핑의 부정확성 제거. 한 번 실행하면 이후 모든 검색에서 정확한 페이지 번호
- **난이도**: 하 (기존 `buildPageMapping` 로직을 Python으로 실행 후 마커 삽입)

### D3. 배치 처리 CLI
- **현재**: 분석 작업은 `add_fullpage.py`를 수동 실행
- **제안**: `python3 searchinmd.py --input keywords.xlsx --docs ./ncs/ --output results.xlsx --dashboard` 형태의 CLI
- **가치**: 커맨드 한 줄로 키워드 검색 → 등급 분류 → Excel 저장 → 대시보드 생성 완료
- **난이도**: 중 (기존 로직을 CLI로 통합)

## 우선순위 추천

| 순위 | 기능 | 이유 |
|:----:|------|------|
| **1** | D2. 페이지 마커 자동 삽입 | 난이도 하, TODOS에 이미 있음, 페이지 정확도 즉시 개선 |
| **2** | C2. 대시보드 Excel 업로드 | 난이도 중, 분석할 때마다 코드 수정 불필요, 재사용성 극대화 |
| **3** | B1. 등급 분류 정확도 개선 | 난이도 중, 형식적 언급 vs 실질 내용 구분이 연구 핵심 |
| **4** | A1. 복합 키워드 검색 | 난이도 중, 연구자 요구가 높은 기능 |
| **5** | C1. 자동 보고서 생성 | 난이도 중, 연구 산출물 직접 기여 |
| **6** | D3. 배치 처리 CLI | 난이도 중, 반복 작업 자동화 |
